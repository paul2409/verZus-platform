# M6.1 Competition Discovery Contract

## Purpose

The Compete route must immediately answer:

- Which competition is featured?
- Which competitions are live, upcoming or already entered?
- What is the prize, format, capacity and timing?
- What is the player already registered for?
- How does the competition journey work?

## Approved composition

Desktop:

1. Compete heading and status summary
2. Featured championship hero
3. Discover → Enter → Compete → Earn journey
4. All competitions toolbar
5. Dense competition rows
6. Quick filters, current entry and guide rail

Mobile:

1. Compact heading
2. Featured hero
3. Journey
4. State tabs
5. Competition cards
6. Existing App Shell bottom navigation

## M6.1 boundary

This stage uses typed mock view models. It does not create production API clients, detail routes or entry mutations. Those belong to M6.3–M6.5.

## Visual authority

The approved M6 image is the visual authority. The active retro-competitive tokens remain the implementation authority for color, typography, focus and geometry.
