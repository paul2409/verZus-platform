# M8 — Eight Controlled Stages

1. **M8.1 — Responsive foundation and approved screen**
   - Weekly player, weekly pool, game lane, Crew championship and combine modes.
   - Separate mobile compact list and desktop full table.
   - Current-player pinned position and local visual controls.
2. **M8.2 — Search, filters, sorting, pagination and URL state**
   - Debounced search, query-string state, deterministic sorting and stable pagination.
3. **M8.3 — Schemas, APIs, adapters and query resources**
   - Zod contracts, domain adapters, TanStack Query and mock/real API parity.
4. **M8.4 — Mode resources and ranking composition**
   - Independent weekly player, pool, game, Crew and combine read models.
5. **M8.5 — Current-player position, movement and update stability**
   - Separate position resource, previous-rank movement and predictable live update ordering.
6. **M8.6 — Reliability and edge states**
   - Loading, empty, stale, error, offline, unauthorized and malformed-row isolation.
7. **M8.7 — Performance, accessibility and failure isolation**
   - Large-list performance, keyboard/table semantics, partial failure and controlled refresh.
8. **M8.8 — Testing, observability and release**
   - Unit, integration, E2E, visual regression, telemetry, preview, immutable artifact and rollback.
