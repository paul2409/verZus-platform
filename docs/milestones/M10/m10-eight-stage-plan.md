# M10 — Rewards and Progression: Eight-stage plan

<!-- VERZUS M10.1 -->

1. **M10.1 — Rewards foundation and approved 390px overview**  
   Establish the domain vocabulary, mobile hierarchy, deterministic fixtures and local artwork.
2. **M10.2 — Reward inventory and complete state presentations**  
   Add all, claimable, locked, claimed, expired and revoked inventory views with filters and details.
3. **M10.3 — Schemas, APIs, adapters and query resources**  
   Split progress, inventory, history and achievement resources through Zod, adapters and TanStack Query.
4. **M10.4 — Idempotent reward claim flow**  
   Add server-authoritative eligibility, expected-version checks, idempotency keys and retry-safe confirmation.
5. **M10.5 — Progression track and season progress**  
   Add progression milestones, season summaries, reward thresholds and independently cached widgets.
6. **M10.6 — Achievement detail and auditable reward history**  
   Add achievement details, reward provenance, pagination, expiration and revocation presentations.
7. **M10.7 — Reliability, failure isolation and observability**  
   Add stale, offline, partial-failure, malformed-data and claim-failure handling with request IDs and telemetry.
8. **M10.8 — Release readiness and rollback**  
   Add approved responsive references, accessibility review, optional browser checks, feature isolation and immutable packaging.
