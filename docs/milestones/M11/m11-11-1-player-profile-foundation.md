# M11.1 — Player Profile Foundation

<!-- VERZUS M11.1 -->

## Purpose

Establish the own-player profile route using the approved mobile-first hierarchy before profile mutations or public-player permissions are introduced.

## Screen contract

The first viewport answers:

- Who is this player?
- Is the identity verified?
- What is the player's current competitive position?
- Which Crew and game identities are connected?
- What is the player's recent form?
- Is the player available to compete?

## Included modules

- Identity banner and avatar fallback
- Display name, handle, title, bio, location and visibility
- Matches, win rate, rating, points, rank, streak and trust
- Crew identity
- Game identities
- Recent verified matches
- Achievement preview
- Availability

## Deferred to later M11 stages

- Public profile permission policy
- Edit mutations and image uploads
- Full match-history pagination
- Full achievement and trust history
- Privacy controls
- Suspended and blocked profile screens
- Production API resources

## Responsive rule

The 390px composition is authoritative for M11.1. Tablet and desktop enhance spacing and columns without changing mobile information priority.
